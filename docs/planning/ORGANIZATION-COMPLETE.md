## 📝 Maintenance Guidelines

### After Each Sprint

1. Create `sprints/sprintN/` directory
2. Move sprint docs to `sprints/sprintN/`
3. Create README.md in sprint directory
4. Update PROJECT-STATUS.md

### When Archiving

1. Move to `archive/` directory
2. Update links in active docs
3. Document reason for archiving in archive/README.md

### Never Archive

- Core planning docs (prd.md, FINAL-BLUEPRINT.md, etc.)
- Technical specs (prd-enhancements.md, reports-master-list.md, etc.)
- Current sprint documentation (until sprint is complete)
